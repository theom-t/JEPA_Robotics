# V1.5 High-Acuity Fine-Tuning Strategy (Resolution Interpolation)

## 1. The Challenge: Precision in Robotics
While the V1 foundation run (256x256 resolution, `patch_size=16`) is excellent for learning the macro-physics of the environment, high-frequency robotic tasks like **pincer rotation (Rot)** and **gripper actuation (Grp)** require sub-millimeter visual precision. 

A 16x16 patch on a 256x256 image can occasionally be too "blurry" to determine the exact angle of a small articulated joint. To achieve state-of-the-art robotic control, the Latent World Model needs 4x the spatial acuity.

## 2. The Bottleneck: Patch Dimensionality
The intuitive solution is to simply shrink the `patch_size` from 16 to 8. However, this mathematically breaks the pre-trained weights:
- **`patch_size=16`**: 16x16x3 = 768 pixels. The Patch Projection Layer is a `[768, 256]` matrix.
- **`patch_size=8`**: 8x8x3 = 192 pixels. The Patch Projection Layer becomes `[192, 256]`.

Switching patch sizes would require re-initializing the crucial Patch Projection matrix from scratch, feeding "alien" untrained embeddings into the Transformer blocks and destroying the V1 foundation.

## 3. The ViT Hacker Solution: Resolution Fine-Tuning
Instead of shrinking the patches, we **double the input image resolution** while maintaining the `patch_size`. 

**The Math (V1 ➔ V1.5):**
* **V1 (256x256, Patch 16):** Sequence Length = (256/16)² = 16x16 = 256 patches.
* **V1.5 (512x512, Patch 16):** Sequence Length = (512/16)² = 32x32 = 1024 patches.

**The Benefits:**
1. **100% Core Weight Reuse:** Because a 16x16 patch still contains 768 pixels, our `[768, 256]` projection matrix and all subsequent Attention Blocks remain perfectly intact.
2. **4x the Acuity:** The network instantly receives 1024 high-definition patches of the scene, matching the theoretical precision of an 8x8 patch run without breaking the weights.

## 4. Execution Plan (Post-V1)
Once the massive 100-epoch V1 run concludes, we will execute a short "High-Acuity Burst" (V1.5) that implements a triple-upgrade strategy, heavily inspired by Meta FAIR's V-JEPA 2.1:

1. **Positional Embedding Interpolation:** 
   - The pre-trained V1 positional embeddings are dimensioned for a 16x16 grid.
   - We will write a utility script using JAX's bicubic interpolation to smoothly stretch this 2D grid up to a 32x32 grid.
2. **Dataloader Up-res:**
   - Modify the `BridgeDataLoader` and `SO100DataLoader` to output `512x512` tensors instead of `256x256`.
3. **Hardware Constraints ($O(N^2)$):**
   - Because self-attention complexity scales quadratically with sequence length, processing 1024 patches will dramatically increase VRAM usage on the RTX 5090.
   - We will drop the `batch_size` from `128` down to `64` (or `32` if required).
4. **Dense Predictive Loss (V-JEPA 2.1 Upgrade):**
   - We will modify `JEPAPredictor` and `train_step_fn` to predict *all* tokens (both visible context and masked tokens), forcing the network to learn ultra-consistent dense features.
5. **Deep Self-Supervision (V-JEPA 2.1 Upgrade):**
   - We will apply the JEPA loss function at intermediate layers of the Transformer (not just the final output) to ensure early attention blocks pull their weight and learn rich geometric structures.
6. **Fine-Tuning Burst:**
   - Train the system for a short 20-epoch burst. This gives the attention mechanism enough time to recalibrate to the dense 1024-patch sequences and integrates the new V-JEPA 2.1 loss penalties, sharply lowering the Rot/Grp MSE metrics.

## 5. Transition to V2
This V1.5 high-acuity Latent space will serve as the absolute final, frozen visual cortex used to train the Cross-Attention Language-Conditioned Policy Head in V2.
